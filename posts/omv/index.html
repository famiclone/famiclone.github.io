Хранилище файлов из подручных средств. Часть 1

  • #linux
  • #data
  • #media
  • #server
  • #orangepi
Dec 5, 2019

Сразу предупреждаю “Подручные средства” в заголовке всё таки требуют определенных вложений. Я использовал это словосочетание исключительно потому, что у меня все компоненты просто лежали без дела и я за пару часов собрал из них сабж.

Давно возникло желание иметь девайс к которому можно будет подключится любым устройством дома. Чтобы шарить медиа и прочие файлы между устройствами без задней мысли. Так как у меня два ноутбука (один в качестве десктопа), а второй использую как, собственно, ноутбук (ношу с собой и т.д.). На обоих стоят SSD на 120GB, что как-бы маловато в 2019. Плюс еще к этому разные системы (на декстопе macOS, на втором ноуте - ArchLinux), а это разные файловые системы.

Необходимый минимум

  • Orange PI ( В моём случае Orange PI PC )
  • 2.5” HDD / SSD
  • Блок питания
  • Карта памяти от 8GB (10 класса) и выше
  • USB 3.0 Бокс для жёсткого диска

Первый шаг

С самого начала нужно будет записать образ на карту памяти. Образ нужен для конкретного одноплатника, поэтому ищем по моделям здесь и качаем Armbian Buster (это важно). Скачали образ? Теперь переходим к записи. Для этого есть очень удобная утилита - Etcher. Здесь всё максимально просто:

  1. Выбор образа
  2. Выбор носителя для записи
  3. Кнопка старта записи

После успешной записи вставляем карту в одноплатник и подключаем к нему блок питания и кабель ethernet. Ethernet должен быть подключен к свободному порту на роутере.

Подключение по ssh

Итак, одноплатник работает (это должно быть видно по светодиодам на плате), теперь нужно узнать его IP-адрес в локальной сети. Для этого, идём в веб-интерфейс своего wifi-роутера. В зависимости от модели и производителя роутера, интерфейс будет отличаться. Короче говоря нам нужен раздел DHCP, в ней будет таблица со списком всех подключенных устройств.

Здесь то и можно будет увидеть название нашего одноплатника и присвоеный ему IP-адрес. Переходим в терминал (macOS, linux, freebsd и прочие unix-like системы) (или в Windows PowerShell) и вводим:

ssh root@<ваш_ip_адрес>

Далее нужно будет ввести пароль 1234. Если пароль принят, то появится приветствие примерно такого вида:

На этом этапе требуется изменить текущий пароль 1234 на более сложный пароль и создать нового пользователя. Когда пользователь создан, нужно будет выйти и зайти под ним.

exit
ssh <имя_нового_пользователя>@<ваш_ip_адрес>

Теперь можно переходить к обновлению системы:

sudo apt update && sudo apt upgrade

После успешного обновления, заходим в armbian-config

sudo armbian-config

Далее проследовать в Software -> Softy и отметить клавишей “Пробел” нужные пакеты

  • Samba
  • OMV

После этого выбрать INSTALL и дождаться окончания установки.

Она может занять до 30 минут, и может показаться, что процесс висит. Но лучше всего не прерывать процесс и дождаться конца установки.
